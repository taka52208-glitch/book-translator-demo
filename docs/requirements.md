# 洋書自動翻訳システム デモ版 - 要件定義書

> 要件定義の作成原則
> - **「あったらいいな」は絶対に作らない**
> - **拡張可能性のための余分な要素は一切追加しない**
> - **将来の「もしかして」のための準備は禁止**
> - **今、ここで必要な最小限の要素のみ**

## 1. プロジェクト概要

### 1.1 成果目標
案件獲得のため、Kindle洋書→スクリーンショット→ChatGPT翻訳→Notion保存の一連フローを実演できるデモシステムを構築し、技術力と実現可能性を証明する。

### 1.2 成功指標

#### 定量的指標
- 翻訳精度: ページ跨ぎ文章の正確な接続率 95%以上
- 処理速度: 1ページあたり30秒以内で翻訳完了
- デモ実演: 10ページ以上の連続処理をエラーなく完遂
- コスト試算: 250枚×月3回の運用コストを明確に提示可能

#### 定性的指標
- クライアントが「これなら任せられる」と感じる完成度
- 非技術者でも理解できるシンプルな操作フロー
- 翻訳結果が読みやすく整理されている
- スクリーンショット自動化の代替案を提示できる

## 2. システム全体像

### 2.1 主要機能一覧
- **翻訳デモ機能**: Kindleスクショをアップロードして翻訳結果を即座に表示
- **ワークフロー説明機能**: n8n連携フローとコストを視覚的に説明

### 2.2 ユーザーロールと権限（MVP版）
- ゲスト: 全機能にアクセス可能（認証不要）

### 2.3 認証・認可要件
- 認証方式: なし（デモ版のため不要）
- セキュリティレベル: 公開情報のみ
- 管理機能: 不要

## 3. ページ詳細仕様

### 3.1 P-001: 翻訳デモ画面

#### 目的
クライアントが翻訳品質とページ跨ぎ対応を実際に体験し、技術力を確認できる

#### 主要機能
- Kindleスクショ画像のドラッグ&ドロップアップロード
- 前ページ末尾（英文）の入力欄（ページ跨ぎ対応デモ用）
- OpenAI APIによる翻訳実行
- 翻訳結果のリアルタイム表示
- [IMAGE_HERE]マーカーの視覚的表示
- 原文末尾（次ページ用）の表示

#### 必要な操作
| 操作種別 | 操作内容 | 必要な入力 | 期待される出力 |
|---------|---------|-----------|---------------|
| アップロード | スクショ画像を送信 | 画像ファイル（JPG/PNG） | プレビュー表示 |
| 入力 | 前ページ末尾を入力 | 英文テキスト（任意） | 入力欄に表示 |
| 翻訳実行 | 翻訳ボタンクリック | なし | 翻訳結果表示 |

#### 処理フロー
1. ユーザーがスクショ画像をドラッグ&ドロップ
2. （任意）前ページ末尾の英文を入力
3. 「翻訳する」ボタンをクリック
4. バックエンドがOpenAI APIに画像+プロンプトを送信
5. 翻訳結果を画面に表示
6. 原文末尾があれば「次ページ用」として表示

#### データ構造（概念）
```yaml
翻訳リクエスト:
  画像: Base64エンコード画像データ（必須）
  前ページ末尾: 英文テキスト（任意）

翻訳レスポンス:
  翻訳本文: 日本語テキスト（必須）
  原文末尾: 英文テキスト（任意、ページ途中終了時）
  ページ情報: ページ番号文字列（任意）
  イラスト位置: [IMAGE_HERE]マーカーの有無
```

## 4. データ設計概要

### 4.1 主要エンティティ

```yaml
翻訳リクエスト:
  概要: 1回の翻訳処理に必要な入力データ
  主要属性:
    - 画像データ（Base64）
    - 前ページ末尾テキスト
  関連: なし（ステートレス）

翻訳レスポンス:
  概要: 翻訳処理の結果データ
  主要属性:
    - 翻訳本文
    - 原文末尾
    - ページ情報
  関連: なし（ステートレス）
```

### 4.2 エンティティ関係図
```
[画像アップロード] → [翻訳リクエスト] → [OpenAI API] → [翻訳レスポンス] → [画面表示]
```

### 4.3 バリデーションルール
```yaml
画像ファイル:
  - ルール: JPG/PNG形式、10MB以下
  - 理由: OpenAI Vision APIの制限対応

前ページ末尾:
  - ルール: 500文字以下の英文
  - 理由: 過度に長いテキストの防止
```

## 5. 制約事項

### 外部API制限
- **OpenAI API**: レート制限あり（TPM/RPM制限）
- **画像サイズ**: OpenAI Vision APIは20MB以下推奨

### 技術的制約
- **ステートレス設計**: MVP版はセッション間でデータを保持しない
- **同時実行**: 1リクエストずつ順次処理

## 5.1 セキュリティ要件

### 基本方針
本プロジェクトは **CVSS 3.1（Common Vulnerability Scoring System）** に準拠したセキュリティ要件を満たすこと。

### MVP判定

**認証機能がない場合（本プロジェクト）**:
- ブルートフォース対策は不要
- 基本的なセキュリティヘッダーのみ設定

**その他の一般要件**:
- ✅ HTTPSの強制（本番環境）
- ✅ セキュリティヘッダー設定（本番環境）
- ✅ 入力値のサニタイゼーション
- ✅ エラーメッセージでの情報漏洩防止
- ✅ APIキーの環境変数管理

### 運用要件：可用性とヘルスチェック

**ヘルスチェックエンドポイント（必須）**:
- エンドポイント: `/api/health`
- 目的: Cloud Runでのliveness/readinessプローブ
- 要件: 5秒以内の応答

**グレースフルシャットダウン（必須）**:
- SIGTERMシグナルハンドラーの実装
- 進行中のリクエスト完了まで待機
- タイムアウト: 8秒

## 6. 複合API処理（バックエンド内部処理）

### 複合処理-001: 画像翻訳処理
**トリガー**: 「翻訳する」ボタンクリック
**フロントエンドAPI**: POST /api/translate
**バックエンド内部処理**:
1. 画像データのBase64デコード・検証
2. プロンプト構築（前ページ末尾があれば追加）
3. OpenAI Vision APIへリクエスト送信
4. レスポンス解析（翻訳本文・原文末尾・ページ情報の抽出）
5. 結果をJSON形式で返却
**結果**: 翻訳結果JSON
**外部サービス依存**: OpenAI API
